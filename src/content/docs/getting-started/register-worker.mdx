---
title: Register a Worker
description: Connect a machine to your CropsCode orchestrator as a worker.
---

A worker is a machine running Podman that executes containers on behalf of the orchestrator. Registration is a one-time step that gives the machine a token to authenticate its WebSocket connection.

## Prerequisites

- [CLI installed](/getting-started/install-cli) and authenticated (`cropscode login`)
- **Podman** installed and running on this machine
- Network access to the orchestrator (outbound only — no inbound ports needed)

## Quick path

```bash
cropscode onboard
```

The onboard wizard checks everything (auth, worker registration, orchestrator reachability, container runtime, disk/memory) and fixes issues automatically.

## Manual path

### Register

```bash
cropscode worker --register
```

This:
1. Registers the machine with the orchestrator
2. Stores a worker token in `~/.cropscode/worker.json`
3. Associates the worker with your tenant

### Start

```bash
cropscode worker
```

The worker connects outbound to the orchestrator via WebSocket and begins accepting container commands.

### Verify

```bash
cropscode status
```

You should see:

```
User:         you@example.com
Worker:       registered (online)
Tenant:       your-tenant
Orchestrator: connected
```

The worker also appears in the orchestrator dashboard under the Workers panel.

## Health check

Run `cropscode doctor` at any time to verify the worker's health:

```bash
cropscode doctor
```

It checks: authentication, worker registration, orchestrator connectivity, container runtime (Podman), disk space, and available memory.

## Multi-machine setup

Repeat these steps on each machine you want to connect. Workers connect outbound — no inbound ports needed, works behind NAT and firewalls.
